Use case for BEM Plugin?

The BEM Block Element Modifier Pattern

The BEM-Pattern is a CSS naming convention and pattern to structure code via css class names
  • Block correlates to components
  • Element correlates to items
  • Modifier correlates to states
Based on this, the BEM Plugin adds special CSS classes to components following a special rules.
The BEM Layouts provide predefined CSS classes following this rule.

With the help of the BEM Plugin and these predefined classes, you can customize space(s) between components.
To add space between components
1. You should first apply the BEM Plugin to the outmost container, e.g. to the Panel
<Panel>
  <layout>
    <layout_HBox/>
  </layout>
  <items>
    <Container width="25%"/>
    <Container width="25%"/>
    <Container width="25%"/>
  </items>
  <plugins>
    <ui:BEMPlugin block="{BEMLayouts.HORIZONTAL_SPACING_BLOCK}"/>
  </plugins>
  <dockedItems>
    <Container width="25%"/>
  </dockedItems>
</Panel>
But this has <no visible effect>. It <only attaches a certain class>, here {BEMLayouts.HORIZONTAL_SPACING_BLOCK} to the Panel.
2. Now, there are three different ways to get space between your components. I.e. to attach another CSS class which causes the spacing
The first approach applies space only between these items which are directly targeted: 
<Panel>
  <layout>
    <layout_HBox/>
  </layout>
  <items>
    <Container width="25%">
      <mixins>
        <ui:BEMMixin bemElement="{BEMLayouts.HORIZONTAL_SPACING_ELEMENT_ITEM}"/>
      </mixins>
    </Container>
    <Container width="25%">
      <mixins>
        <ui:BEMMixin bemElement="{BEMLayouts.HORIZONTAL_SPACING_ELEMENT_ITEM}"/>
      </mixins>
    </Container>
    <Container width="25%"/>
  </items>
  <plugins>
    <ui:BEMPlugin block="{BEMLayouts.HORIZONTAL_SPACING_BLOCK}"/>
  </plugins>
  <dockedItems>
    <Container width="25%"/>
  </dockedItems>
</Panel>
And yes! Both containers have to be extended.

The second approach is to apply space to all items and only items! This is done via the following directive: 
<Panel>
  <layout>
    <layout_HBox/>
  </layout>
  <items>
    <Container width="25%"/>
    <Container width="25%"/>
    <Container width="25%"/>
  </items>
  <plugins>
    <ui:BEMPlugin block="{BEMLayouts.HORIZONTAL_SPACING_BLOCK}"/>
  </plugins>
  <dockedItems>
    <Container width="25%"/>
  </dockedItems>
  <defaults>
    <Component>
      <mixins>
        <ui:BEMMixin bemElement="{BEMLayouts.HORIZONTAL_SPACING_ELEMENT_ITEM}"/>
      </mixins>
    </Component>
  </defaults>
</Panel>
The third approach is to apply space to all items, including dockedItems. This is done via the attribute defaultElement: 
<Panel>
  <layout>
    <layout_HBox/>
  </layout>
  <items>
    <Container width="25%"/>
    <Container width="25%"/>
    <Container width="25%"/>
  </items>
  <plugins>
  <ui:BEMPlugin block="{BEMLayouts. HORIZONTAL_SPACING_BLOCK}"
                defaultElement="{BEMLayouts.HORIZONTAL_SPACING_ELEMENT_ITEM}"/>
  </plugins>
  <dockedItems>
    <Container width="25%"/>
  </dockedItems>
</Panel>

  • The examples demonstrate the usage of the HORIZONTAL spacing. You could also space VERTICAL.
  • You can add much more space by adding the attribute modifier="{BEMLayouts.HORIZONTAL_SPACING_MODIFIER_200}
    or modifier="{BEMLayouts.HORIZONTAL_SPACING_MODIFIER_300} to the ui:BEMPlugin. I.e. 
    <plugins>
  <ui:BEMPlugin block="{BEMLayouts.HORIZONTAL_SPACING_BLOCK}"
                modifier="{BEMLayouts.HORIZONTAL_SPACING_MODIFIER_200}"/>
</plugins>
