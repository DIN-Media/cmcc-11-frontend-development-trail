Candy Sites

A candy host returns the favor, i.e. it redirects the callers request to the caller itself. This allows the developer to start his webapps locally using a maven plugin and work with portless https URLs, without having the need to start and configure any reverse proxy such as apache or nginx. In a Commerce Led scenario, the candy host will redirect the fragment calls to the developers workstation. This way many developers can share the same Commerce installation without interference. The candy setup also works locally with the docker setup, but only if you choose the Hostname option in the compose script.
